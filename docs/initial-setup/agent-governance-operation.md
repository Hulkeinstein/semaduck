## 1. 계층적 원칙 적용 (Hierarchical Principles)

우리는 시스템의 호환성과 프로젝트의 아키텍처를 동시에 만족시키기 위해 다음의 우선순위 원칙에 따라 자산을 운영합니다.

### 🥇 [Priority 1] Antigravity 시스템 원칙: `.agent/`
*   **대상**: 시스템이 직접 인식하고 가동해야 하는 **규칙(Rules)** 및 **워크플로우(Workflows)**.
*   **위치**: 
    - `.agent/rules/`: 디자인 시스템, 코드 컨벤션, API 설계 규칙.
    - `.agent/workflows/`: 메인 구축 프로세스, 예약 시스템 구축 절차.
*   **이유**: Antigravity의 슬래시 명령어 및 자동화 기능을 100% 활용하기 위한 최우선 원칙입니다.

### 🥈 [Priority 2] 프로젝트 지침 원칙: `directives/`
*   **대상**: 시스템 운영 외의 특정 비즈니스 로직이나 가이드라인인 **지침(Directives/SOP)**.
*   **위치**: 
    - `directives/`: 웹 리서치 절차, 환경 설정 가이드 등.
*   **이유**: 사용자님이 설계하신 3계층 아키텍처의 **Layer 1 (Directive)** 정체성을 유지하며, 업무 매뉴얼로서의 독립성을 가집니다.

---

## 2. 작업 프로세스 (Work-flow)

작업 중 지침이나 규칙이 변경될 경우, 다음의 루프를 따릅니다.

1.  **수정 (Edit)**: 모든 지침의 수정은 반드시 **`directives/`** 폴더 내의 파일에서 이루어집니다.
2.  **동기화 (Sync)**: 수정 완료 후, 에이전트(저)는 해당 내용을 **`.agent/workflows`** 또는 **`.agent/rules`** 폴더에 즉시 복사하여 동기화합니다.
3.  **검증 (Validate)**: 수정된 지침이 Antigravity 시스템에서 정상적으로 인식되는지 확인합니다.

---

## 3. 에이전트의 책임 (Agent Responsibilities)

*   **동기화 보장**: 저는 `directives/`와 `.agent/` 간의 일관성을 상시 유지할 책임이 있습니다.
*   **시스템 호환성 유지**: Antigravity 시스템 도구가 항상 최신 버전의 우리 SOP를 참조할 수 있도록 관리합니다.
*   **원칙 고수**: 사용자님이 `AGENTS.md`에서 정의하신 'Layer 1: Directive'의 정체성을 훼손하지 않으면서 시스템을 운영합니다.

---

## 4. 기대 효과

*   **아키텍처 선명도**: 프로젝트 외부에서 보았을 때 `directives/`라는 폴더를 통해 에이전트의 지능적 통제 체계가 명확히 보입니다.
*   **강력한 호환성**: Antigravity의 강력한 워크플로우 UI와 규칙 검사 기능을 그대로 활용할 수 있습니다.
*   **유연한 확장**: 다른 AI 환경으로 프로젝트가 이동하더라도 `directives/` 폴더만 있으면 동일한 에이전트 지능을 복구할 수 있습니다.
